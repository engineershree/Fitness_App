# Database Setup Guide for Team Members

## 🚀 Quick Start for Your Teammate

### Files to Share
1. **`fitness_database_schema.sql`** - Complete database structure (tables, indexes, relationships)
2. **`fitness_database_schema.json`** - Detailed schema information
3. **`fitness_sample_data.json`** - (Optional) Sample data for testing

### 📋 What Your Teammate Needs

#### 1. Install PostgreSQL
```bash
# Download from: https://www.postgresql.org/download/windows/
# Or use package manager:
# Chocolatey: choco install postgresql
# Homebrew (Mac): brew install postgresql
```

#### 2. Create Database
```sql
-- Connect to PostgreSQL and create database
CREATE DATABASE fitness_app;
CREATE USER fitness_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE fitness_app TO fitness_user;
```

#### 3. Import Schema
```bash
# Using psql command line
psql -U fitness_user -d fitness_app -f fitness_database_schema.sql

# Or using pgAdmin GUI
# Right-click on database -> Restore -> Select SQL file
```

#### 4. Update Environment Variables
Create `.env` file in project root:
```env
DATABASE_URL=postgresql://fitness_user:your_password@localhost:5432/fitness_app
```

## 🔧 How to Generate Schema Files

### Method 1: Use the Python Script (Recommended)
```bash
cd Fitness_App
python database_export.py
```

This creates:
- `fitness_database_schema.sql` - SQL commands to recreate tables
- `fitness_database_schema.json` - Detailed schema documentation
- `fitness_sample_data.json` - Sample data (optional)

### Method 2: Manual pg_dump (if PostgreSQL tools installed)
```bash
# Schema only (no data)
pg_dump -U your_username -h localhost -d fitness_app --schema-only -f fitness_schema.sql

# Schema with limited sample data
pg_dump -U your_username -h localhost -d fitness_app --data-only --rows-per-insert 10 -f fitness_sample.sql
```

## 🛡️ Security Best Practices

### ✅ Safe to Share
- Database schema (table structures)
- Column definitions
- Relationships and indexes
- Sample data (limited, non-sensitive rows)

### ❌ Never Share
- Production database with real user data
- Environment files (.env)
- Database connection strings with passwords
- User personal information

### 🔒 Recommended Approach
1. **Share schema only** - No actual data
2. **Use sample data** - Limited rows for testing
3. **Sanitize data** - Remove any sensitive information
4. **Change passwords** - Use different credentials for development

## 🔄 Keeping Schema in Sync

### When Schema Changes
1. Update SQLAlchemy models in `app/models/`
2. Run `python database_export.py` to generate new schema
3. Share updated schema files with team
4. Teammate updates their database

### Version Control
- ✅ Commit `database_export.py` to git
- ✅ Commit schema files to git (if you want to track changes)
- ❌ Never commit `.env` files
- ❌ Never commit database backups with real data

## 🐛 Troubleshooting

### Common Issues
1. **"Connection refused"** - PostgreSQL not running
2. **"Database doesn't exist"** - Create database first
3. **"Permission denied"** - Check user permissions
4. **"SQL syntax error"** - Make sure PostgreSQL version matches

### Commands to Check Setup
```bash
# Check PostgreSQL is running
pg_isready

# List databases
psql -l

# Connect to database
psql -U fitness_user -d fitness_app

# List tables
\dt
```
